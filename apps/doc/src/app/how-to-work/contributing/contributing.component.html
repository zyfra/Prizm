Требования к заголовку PR, Issue

Заголовок сообщения коммитов и PR должны состоять из:

<type>(<scope>/(component?)): <short summary>

type

Тип коммита

Должен быть одним из:

  1. build: Изменения в способе сборки, например nx
  2. ci: Изменения в pipeline .gitlab-ci.yml/li>
  3. docs: Документация
  4. feat: Новая фича
  5. fix: Фикс бага
  6. perf: Повышение производительности
  7. refactor: Изменения, которые не исправляют баги и не добавляют новую фичу, а лишь изменяют качество кодовой базы.
  8. test: Добавление теста

scope

Окружение npm библиотека или же приложение

Должны быть одним из:

  1. core
  2. components - КБ
  3. charts
  4. helpers - библиотека хелперов
  5. schematics - библиотека схематиков, установщиков
  6. executors - инструменты для самой работы sdk
  7. generators - схематики для самой sdk
  8. doc - приложение витрины компонентов

component

Часть непосредственно самого scope.

Для components может быть:

`button`, `calendar`, `input` и т.д.

Информация об установке компонента

Examples

Именование:

<component>-basic-example - базовый пример без лишней конфигурации

Naming

Модули, компоненты, директивы, pipe и все что выступает в public через {{ importing }} Должны иметь префикс - prizm

Пример: PrizmInputModule

Cтили публикуемые наружу (styles.less).

Стили не должны попадать в глобальную область видимости. ViewEncapsulation - всегда Emulated (то что по дефолту).

Subscribers

Icons

Форматы

В базе имеются форматы иконок: 24, 16,

24 - базовый

16, 8 - дополнительные, специально перерисованы для случаев когда scale иконки не удачный вариант.

Требования к иконкам

  1. Иконки должны быть с квадратной рамкой
  2. Отцентрированы
  3. Изображение не выходит за рамки квадрата
  4. Для экспорта изображение подготовлено в формате path а не линии.
  5. Fill-rule правильно обозначен (https://github.com/jaywcjlove/svgtofont/issues/79) прогнан через плагин фигмы Fill Rule Editor plug-in,

Как добавить новую иконку в Prizm:

  1. Экспортировать иконку из Figma в формате svg
  2. Положить в SDK /libs/components/assets/icons
  3. Выполнить сборку шрифтов nx run components:build-icons
